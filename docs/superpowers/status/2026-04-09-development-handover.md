# Avoid-Nav Beijing：开发进度与技术架构总结 (2026-04-09)

## 1. 项目概况
构建了一个基于高德地图 JS API 2.0 的北京避让导航工具，旨在通过 5 轮递归探测技术，彻底规避路径上的单向/双向交通监控风险点。

## 2. 核心技术栈
- **前端**：Next.js 14 (App Router) + TailwindCSS v4 + Lucide Icons。
- **地图**：AMap JS API 2.0 (使用 `LabelsLayer` 进行 WebGL 万级点位渲染)。
- **存储与索引**：Dexie.js (IndexedDB) 离线存储，RBush (R-Tree) 实现毫秒级空间检索。

## 3. 核心算法逻辑

### 3.1 5 轮递归规避 (5-Stage Recursive Avoidance)
由于高德地图的规避区域 (`avoidpolygons`) 是权重的，且绕路后可能产生新风险，系统执行以下循环：
1.  **探测**：获取当前路径，扫描路径周边 80 米内的风险点。
2.  **过滤**：通过“方向感知算法”剔除安全点。
3.  **累加**：将新发现的冲突点加入“黑名单池”。
4.  **冷却**：强制等待 1200ms (规避高德 QPS 限制 10021 错误)。
5.  **重绘**：将黑名单转换为 100m 见方的规避方块，重新发起规划。
*注：最多尝试 5 次，确保路径最终“干净”。*

### 3.2 拓扑方向匹配 (Directional Filtering)
- **语义提取**：通过正则从点位名称中提取“西向东”、“双向”等词汇。
- **角度匹配**：计算路径在该点位的行驶向量角度 $\theta_{car}$，并与监控方向 $\theta_{risk}$ 对比。
- **判定规则**：夹角 $|\theta_{car} - \theta_{risk}| \le 75^\circ$ 视为冲突，否则忽略（如反向行驶或横穿）。

### 3.3 坐标安全保障
- **零 NaN 容忍**：所有传给 API 的坐标必须通过 `isSafe` 校验（检查 `isNaN` 及北京地理边界）。
- **精度裁剪**：强制 `toFixed(6)` 减少 URL 长度，防止 `413 Request Entity Too Large` 错误。

## 4. UI 交互功能
- **左侧：风险管控中心**
    - 实时显示所有已探测到的冲突点。
    - **手动接管**：点击“眼睛”图标可强制忽略或避让特定点，路线实时重绘。
- **右侧：算法调试控制台**
    - 实时打印递归轮次、冲突点录入、以及方向忽略日志（行驶角度 vs 监控角度）。
- **搜索**：集成 AutoComplete，支持地点联想。

## 5. 待办事项 (Next Steps)

3.  **模式预设**：增加“进京证模式”、“货车模式”等一键过滤规则。

## 6. 环境配置要求
- `.env.local` 必须包含：
    - `NEXT_PUBLIC_AMAP_KEY`
    - `NEXT_PUBLIC_AMAP_SECURITY_JS_CODE` (32位安全密钥)
- 运行 `npm run preprocess` 更新精炼数据。
