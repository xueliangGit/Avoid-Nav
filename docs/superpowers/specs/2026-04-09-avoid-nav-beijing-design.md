# 技术设计文档：Avoid-Nav Beijing (北京避让导航)

## 1. 项目背景与目标
针对北京复杂的交通限行、监控及政策规则，利用高德地图 API 为驾驶者提供一个精准的“合规避让”工具。通过处理 11w+ 风险点位，实现静态路径规划避让与动态行驶实时预警的双重安全保障。

## 2. 核心业务流程
1.  **自动感知**：启动应用即获取用户定位，展示周边 500m 范围内的风险点。
2.  **智能预设 (Mode C)**：提供“外地牌”、“京牌”、“货车”等预设档位，自动映射风险等级。
3.  **精准导航 (Mode B)**：
    *   输入终点后，系统尝试规划“零风险”路径。
    *   若路径中包含未避让风险点，开启“实时探测”模式。
4.  **动态交互 (Feedback B)**：行驶中检测到前方 500m 内有风险点，弹出浮窗询问：“发现限行监控，是否绕行？”。

## 3. 技术架构设计

### 3.1 前端与地图引擎
- **框架**：Next.js 14 (App Router)。
- **地图**：高德 JS API 2.0 (使用 `@amap/amap-jsapi-loader`)。
- **高性能渲染**：`AMap.LabelsLayer` (WebGL 渲染 11w+ 点位)。

### 3.2 数据处理与存储
- **离线存储**：**Dexie.js** (IndexedDB) 存储精炼后的点位数据。
- **空间索引**：**RBush (R-Tree)** 驻留内存，负责毫秒级的 `500m` 范围检索。
- **预处理脚本**：`scripts/preprocess.ts` 将原始 JSON 转换为轻量化数组格式。

### 3.3 核心算法逻辑
- **路径抽稀 (RDP Algorithm)**：将高德路径点集压缩，以便在 API 限制（100 个避让区）内进行精准匹配。
- **滑动窗口探测**：
    *   静态：导航启动前搜索路径周边 500m 的风险点并注入 `avoidpolygons`。
    *   动态：行驶中每 3 秒根据经纬度 + 行驶角度生成 500m 探测缓冲区，检索 R-Tree。

## 4. UI/UX 规范
- **预警浮窗 (InteractionOverlay)**：展示风险等级图标、实时距离（500m 递减）、“绕行/忽略”操作。
- **地图视觉**：针对不同 `aa` 类型使用红、蓝、橙、紫四色图标。
- **容错处理**：若无法规划完全避让路径，提示用户并降级为“实时提醒模式”。

## 5. 数据字典映射 (aa 字段)
- `aa: "2"` -> 红色 -> 高危（未办证/限行监控）。
- `aa: "1"` -> 蓝色 -> 限制（高峰期/顺义）。
- `aa: "6"` -> 紫色 -> 政策（六环外进京证）。
- `aa: "3"` -> 橙色 -> 低风险。

## 6. 成功标准
- 11w+ 点位加载时间 < 3s。
- 500m 范围检索延迟 < 50ms。
- 成功规避用户设定的所有风险等级点位。
