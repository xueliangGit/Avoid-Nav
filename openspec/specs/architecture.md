# 技术架构：Avoid-Nav Beijing

## 1. 系统架构
- **前端框架**：Next.js 14 (App Router)。
- **地图引擎**：高德 JS API 2.0。
- **离线存储**：Dexie.js (IndexedDB)。
- **空间计算**：RBush (R-Tree 索引)。

## 2. 核心算法逻辑

### 2.1 海量数据处理与渲染
- **数据源**：`data.json` (11w+ 条)。
- **渲染方案**：使用 `AMap.LabelsLayer`。
    - 将数据按风险等级（`risk: 1/2/3`）预分类。
    - 使用内存中的 `TypedArray` 存储坐标，确保缩放时无 GC 压力导致的卡顿。

### 2.2 动态避让导航算法 (Dynamic Avoidance)
由于高德 API 限制每条路线最多 100 个 `avoidpolygons`，我们将采用滑动窗口策略：
1. 获取初始路径点集 `pathPolyline`。
2. 对 `pathPolyline` 进行抽稀和缓冲区生成（设定 500m 搜索半径）。
3. 调用 **RBush** 在本地数据库中检索位于缓冲区内的、且属于“必避”类型（`aa:2`）的点位。
4. 按距离或重要度选出前 100 个点，注入规避参数，重新规划路径。

### 2.3 数据字典映射 (Map Schema)
根据页面逻辑与 `ico` 原数组定义：
- `aa: "2"` -> 红色图标 -> 高危（未办证+高峰）。
- `aa: "1"` -> 蓝色图标 -> 限制（仅高峰/顺义）。
- `aa: "6"` -> 紫色图标 -> 政策（六环外证即可）。
- `aa: "3"` -> 橙色图标 -> 低风险。

## 3. 部署方案
- 本地开发：Next.js + 自动化更新脚本。
- 部署环境：支持静态导出 (SSG) 或 Vercel 托管。
